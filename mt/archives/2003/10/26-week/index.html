▽[RD-Style] 計画覚書

ネット de ナビは便利なんだが，本体の電源が入ってないと使えないという問題がある． まあ当然と言えばそうなんだが，HDD という駆動系部品が入っているので， 電源つけっぱなしは気分的にあまり良くない．

そこで，この Server に Apache + PHP + PostgreSQL があることを利用して， 予約サーバを作成してみるという計画を立ててみる 変な日本語(+_+;)． で，どんなものかというと，
  • UKTvMail で送信された内容を Database に蓄積する．
  • 一定間隔毎に，RD-X3 が起動しているかどうかを確かめ，起動していれば蓄積された予約を送信する．
  • 送信が成功すれば蓄積された予約のフラグを「送信済み」に変更．
  • Database 内の予約の変更・修正・複製も可能．
  • ようするにライブラリのかわりにもなる．
  • 「毎週予約フラグ」も用意してみる．送信成功時に来週の同時刻の予約を作成する．
てな感じかな．大して大がかりではないと思うんだけど，どうだろうか(+_+;)．

▽[RD-Style] RD-X3 with UKTvMail

うはぁ．．．．便利すぎ．．．． 地上波 iEPG は基本的に ONTV JAPAN を使ってお好みで番組で検索をかけてるんだけど， SkyPerfecTV のがいまいち使いづらい．まあ AT-Xが ないってのが一番の問題だったりするんだけど(ぉ， それがあるところってテレビ王国位しかないんだよね． で，ここの MYPAGE サービスって結構良いんだけど，これはネット de ナビからは使えない． まあ https 使ってるからなんだけど，このままじゃどうしようもないのよね．

で，探すと，iEPG の関連づけで RD-X3 に直接 HTTP 送信してくれる Software がありますた．それが UKTvMail．これだとテレビ王国の予約ボタンをクリックして開くを選ぶと情報が入った状態で起動してくれる． それから編集してOKボタンを押すと RD-X3 に予約がいく． これはイイ(ﾟ∀ﾟ)！．コピペしなくても良いのは非常に便利ですｙｐ！． ますます充実依存度上昇により束縛(ぉ されていく我が環境．この次はいったい何が！！ ダメ人間街道まっしぐら．．．．＿/￣|○

▽[RD-Style] PC との連携

今調べてるのが，PC 中の AVI を DVD-RAM 経由で RD-X3 に持ってく方法． 普通に MPEG2 にすればいいらしいので， 早速TMPGEnc を買ってみることに．いろいろ体験版をさわってみたんだが， やっぱバッチ処理が一番良い感じなのはこれだね． で，480x480 の 4.5Mbps 位で作成して DVD-RAM に VR Format で書き込むといけるらしいのだが，まあそれは TMPGEnc が来てから．

それにして，MPEG2 の Encode ってやっぱ時間かかるのね． DivX 3.11alpha だと Filter 類を全くかけない設定だと， 実時間よりも短くなるんだが，MPEG2 だと実時間程度． まあ仕方ないかな．
