▽[RD-Style] 計画覚書

ネット de ナビは便利なんだが，本体の電源が入ってないと使えないという問題がある． まあ当然と言えばそうなんだが，HDD という駆動系部品が入っているので， 電源つけっぱなしは気分的にあまり良くない．

そこで，この Server に Apache + PHP + PostgreSQL があることを利用して， 予約サーバを作成してみるという計画を立ててみる 変な日本語(+_+;)． で，どんなものかというと，
  • UKTvMail で送信された内容を Database に蓄積する．
  • 一定間隔毎に，RD-X3 が起動しているかどうかを確かめ，起動していれば蓄積された予約を送信する．
  • 送信が成功すれば蓄積された予約のフラグを「送信済み」に変更．
  • Database 内の予約の変更・修正・複製も可能．
  • ようするにライブラリのかわりにもなる．
  • 「毎週予約フラグ」も用意してみる．送信成功時に来週の同時刻の予約を作成する．
てな感じかな．大して大がかりではないと思うんだけど，どうだろうか(+_+;)．
